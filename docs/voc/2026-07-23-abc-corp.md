# VOC: ABC Corp — 다크 모드에서 일부 영역 테마 색상 오류

- **고객사**: ABC Corp
- **접수 경로**: 고객 문의 이메일 (VOC)
- **관련 Task**: https://app.notion.com/p/eecee50412ed4280906825849911feae

## 배경 / 고객 원문 핵심

"다크 모드 사용 시 특정 영역이 라이트 테마 색상으로 남아 가독성이 크게 떨어져"
(환경: Web/Chrome, 다크 모드 ON)

## 현재 문제

Web(Chrome)에서 다크 모드를 활성화했을 때, 일부 UI 영역이 다크 테마 색상으로 전환되지 않고 라이트 테마 색상(흰색 배경, 어두운 텍스트 등)이 그대로 남아 있다.
이로 인해 전체적인 화면 가독성이 저하되며, 고객 경험에 부정적인 영향을 줌.

## 제안 변경사항

1. **재현 확인**: Web(Chrome) 다크 모드 ON 상태에서 각 메뉴/뷰를 순회하며 라이트 색상이 잔존하는 영역 식별
2. **테마 토큰 점검**: 다크 모드 CSS 변수(color token) 미적용 컴포넌트 또는 하드코딩된 색상값 탐색
3. **수정 방향 정리**: 해당 컴포넌트에 다크 모드 스타일 토큰을 올바르게 적용하도록 코드 수정 제안
4. **추가 정보 요청**(필요 시): 고객에게 재현 가능한 정확한 메뉴/뷰, 스크린샷, 워크스페이스 설정 요청

## 우선순위 / 참고

- **체감 우선순위**: Medium — 가독성에 직접 영향을 주는 UI 버그이며, 다크 모드 사용자 전반에 영향 가능
- **판단 근거**: 기능 손상은 아니지만 사용자 경험 저하가 명확하고, 재현 경로가 비교적 단순함 (다크 모드 ON + Chrome)
- **추가 확인 필요**: 정확히 어떤 뷰/메뉴에서 발생하는지 스크린샷 미첨부 상태. 고객 추가 정보 수령 후 재현 범위 확정 필요
